## 🔄 后续优化建议

### 1. 数据对接
- 当前所有新功能（光影雅安、文件上传、地图数据等）均使用前端模拟数据 (Mock Data)。
- 下一步需要对接真实的后端 API，确保数据的实时性和持久化。

### 2. 细节打磨
- **文件上传**: 目前仅实现了 UI 和模拟逻辑，需对接实际的文件存储服务（如 OSS）。
- **地图导航**: 目前使用模拟经纬度，需录入真实的景点和场地坐标数据。
- **富文本展示**: 政策详情和资讯详情目前较为简单，可考虑引入富文本解析器以支持更复杂的排版。

### 3. 用户体验
- 添加更多的加载动画和骨架屏，提升数据加载时的体验。
- 优化图片加载策略（懒加载、缩略图）。

## 🔌 后端接口需求

### 1. 光影雅安 (Films)

#### 1.1 获取影视作品列表
*   **接口**: `GET /api/films/list`
*   **请求参数**:
    *   `page` (int): 页码，从1开始
    *   `size` (int): 每页数量，建议10条
    *   `keyword` (string, 可选): 搜索关键词，支持按作品名称、导演、演员搜索
    *   `type` (string, 可选): 作品类型筛选
        *   可选值: `电影`, `电视剧`, `网络剧`, `微短剧`, `纪录片`
*   **返回数据结构**:
    ```json
    {
      "code": 200,
      "data": {
        "total": 50,
        "list": [
          {
            "id": 1,
            "title": "我的家乡在雅安",
            "type": "电影",
            "director": "张艺谋",
            "actors": "章子怡, 梁朝伟",
            "releaseYear": "2023",
            "poster": "图片URL",
            "description": "简要介绍（列表页显示，约50-100字）",
            "rating": 8.5,
            "tags": ["剧情", "文艺"]
          }
        ]
      }
    }
    ```
*   **字段说明**:
    *   `rating`: 豆瓣评分或平台评分（可选）
    *   `tags`: 作品标签，用于分类展示
    *   `poster`: 作品海报图片URL

#### 1.2 获取影视作品详情
*   **接口**: `GET /api/films/detail`
*   **请求参数**:
    *   `id` (int): 作品ID
*   **返回数据结构**:
    ```json

      "data": {
        "id": 1,
    ```

### 11. 通用响应格式说明

所有接口统一返回格式：
```json
{
  "code": 200,
  "message": "成功",
  "data": {}
}
```

**状态码说明**:
*   `200`: 成功
*   `400`: 请求参数错误
*   `401`: 未授权（需要登录）
*   `403`: 无权限访问
*   `404`: 资源不存在
*   `500`: 服务器内部错误

**分页参数约定**:
*   所有列表接口默认支持分页
*   `page`: 从1开始的页码
*   `size`: 每页条数，默认10条，最大100条
*   返回数据包含 `total` (总条数) 和 `list` (当前页数据)
